Ploinky Cloud Documentation

Overview

Ploinky Cloud is an upcoming serverless runtime platform that enables you to deploy Ploinky agents as Lambda-style functions. It provides a self-hosted, open-source alternative to commercial serverless platforms, giving you complete control over your infrastructure while reducing costs.

Key Benefits

üí∞ Cost Efficient

Pay only for actual compute time. No idle costs, no vendor markup.

üîì No Vendor Lock-in

Fully open-source. Deploy on any infrastructure, migrate anytime.

üöÄ Auto-scaling

Automatically scales from zero to thousands of concurrent executions.

üîÑ CLI Compatibility

Same agent definitions work both locally and in the cloud.

Architecture

Ploinky Cloud transforms your containerized agents into serverless functions accessible via HTTP endpoints.

Components

API Gateway

HTTP endpoint routing and request handling

Function Runtime

Container orchestration and execution

Agent Registry

GitHub-based agent repository system

Scaling Engine

Automatic scaling based on demand

Execution Flow

  1. HTTP request arrives at endpoint
  2. Request routed to appropriate agent
  3. Container spun up if not warm
  4. Agent executes with request data
  5. Response returned to client
  6. Container kept warm or scaled down

Getting Started (Preview)

Note: These instructions are for the upcoming release. The platform is not yet available.

Expected Workflow

# Install Ploinky Cloud CLI
npm install -g ploinky-cloud

# Initialize a new function
ploinky-cloud init my-function

# Deploy an agent as a function
ploinky-cloud deploy --agent python-dev --trigger http

# Get the endpoint URL
ploinky-cloud status my-function
# ‚Üí https://my-function.ploinky.app

# Test the function
curl https://my-function.ploinky.app \
  -X POST \
  -H "Content-Type: application/json" \
  -d '{"message": "Hello from Ploinky Cloud!"}'
